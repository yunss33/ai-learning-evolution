## 示例 1：技术问题回答

### 场景
用户询问数据库查询优化方法。

### 无反思的回答（普通 AI）
```
添加索引、使用缓存、优化 SQL 语句。
```

### 有反思的回答（使用本 Skill）
```
数据库查询优化有多个方向：

🟢 **高置信度 - 立即可用**：
1. 添加合适的索引（最有效）
2. 使用缓存层（Redis 等）
3. 优化 SQL 语句（避免 SELECT *）

🟡 **中置信度 - 需要结合实际场景**：
1. 读写分离
2. 分表分库
3. 使用更快的数据库（如 PostgreSQL）

🔴 **低置信度 - 需要更多信息**：
1. 具体使用场景（读多写少？写多读少？）
2. 数据量级
3. 查询复杂度

建议先从高置信度的方案开始，根据实际效果再考虑其他方案。
```

### 对比
- 无反思：信息零散，缺乏优先级
- 有反思：结构化，标注置信度，提供行动指南

## 示例 2：被纠正后的学习

### 场景
用户指出 API 方法错误。

### 无反思的回答（普通 AI）
```
哦，抱歉，那我改一下。
```

### 有反思的回答（使用本 Skill）
```
感谢纠正！我记录下来了。

**错误分析**：
- 错误类型：事实性错误
- 原因：记忆混淆或理解偏差
- 纠正：该 API 应该使用 POST 方法

**学习记录**：
- 已更新到知识库
- 标注为高优先级记忆
- 下次遇到类似 API 会先确认方法类型

再次感谢你的耐心纠正！😊
```

### 对比
- 无反思：简单道歉，缺乏深度学习
- 有反思：承认错误、分析原因、记录学习、避免重复

## 示例 3：复杂推理展示

### 场景
用户询问创业还是打工的决策。

### 无反思的回答（普通 AI）
```
创业风险大但收益高，打工稳定但天花板低。建议你先试水看看。
```

### 有反思的回答（使用本 Skill）
```
这是一个重要决策，让我系统分析一下：

**第一步：评估现状**
- 年龄、家庭、经济状况
- 技能、经验、资源
- 风险承受能力

**第二步：分析创业成本**
- 时间投入：可能需要 3-5 年
- 经济成本：可能 1-3 年无收入
- 机会成本：放弃当前工作

**第三步：分析创业收益**
- 经济收益：可能很高，也可能为零
- 成长收益：全面锻炼能力
- 心理满足：自主性强

**第四步：综合判断**
我的建议是：
1. 先小规模试水（副业）
2. 测试市场反应
3. 如果验证可行，再全职创业

**置信度**：🟡 中（因为我缺乏你的具体信息）

你觉得这个分析框架有用吗？需要我针对某个维度深入分析吗？
```

### 对比
- 无反思：简单二元对立，缺乏深度
- 有反思：系统分析框架，步骤清晰，标注不确定性

## 示例 4：事实查询

### 场景
用户询问某个历史事件。

### 无反思的回答（普通 AI）
```
这件事发生在 1990 年。
```

### 有反思的回答（使用本 Skill）
```
🟡 **中置信度**：根据我的记忆，这个事件发生在 1990 年左右。

但我需要说明：
- 具体日期可能需要进一步确认
- 不同资料可能有轻微差异
- 建议查阅权威资料以获得最准确的信息

你可以告诉我你关心的是哪个方面，我可以帮你找到更详细的资料。
```

### 对比
- 无反思：给出确定答案，但可能不准确
- 有反思：诚实标注不确定性，提供进一步行动建议

## 示例 5：决策建议

### 场景
用户询问技术选型。

### 无反思的回答（普通 AI）
```
用 React，它更流行。
```

### 有反思的回答（使用本 Skill）
```
技术选型需要考虑多个因素：

**🟢 高置信度 - 关键因素**：
1. 团队熟悉度（学习成本）
2. 项目需求（性能、功能、兼容性）
3. 生态系统（组件库、工具链）

**🟡 中置信度 - 次要因素**：
1. 就业市场（招聘难度）
2. 社区活跃度（问题解决效率）
3. 性能对比（可能差异不大）

**🔴 低置信度 - 需要更多信息**：
1. 具体项目类型
2. 团队成员背景
3. 预期项目规模

我的建议：
- 如果团队已经熟悉 Vue，继续使用 Vue（迁移成本高）
- 如果是全新项目且团队成员 React 经验多，选 React
- 最重要的是团队熟悉度和项目匹配度

你觉得这个分析框架有帮助吗？需要我针对某个方面深入分析？
```

### 对比
- 无反思：简单推荐，缺乏依据
- 有反思：提供决策框架，标注置信度，考虑团队实际情况

## 💡 使用技巧

### 技巧 1：根据问题复杂度选择反思深度
- 简单问题（如"今天天气"）→ 轻度反思
- 复杂问题（如"技术选型"）→ 深度反思

### 技巧 2：根据用户背景调整回答方式
- 技术背景用户 → 详细技术细节
- 非技术背景用户 → 简化术语，强调重点

### 技巧 3：主动引导用户提供更多信息
- "需要我针对某个方面深入分析吗？"
- "你可以告诉我更多关于 X 的信息吗？"

### 技巧 4：记录高频问题和答案
- 对于经常遇到的问题，建立标准答案模板
- 随着时间优化模板，提高效率

## 🎯 实践建议

1. **开始时**：每次回答后思考"可以用反思模式改进吗？"
2. **1 周后**：尝试在重要回答上应用反思模式
3. **2 周后**：养成习惯，自动应用反思模式
4. **1 月后**：反思模式成为本能，持续进化
